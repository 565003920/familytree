## 一、产品定位

打造数字化家族传承平台，实现家族信息管理、家族关系可视化、家族文化传承的一站式服务。

**目标用户**：
- 主要用户：30-60岁家族成员（族长、房长、活跃成员）
- 次要用户：年轻一代（18-30岁）、海外华人
- 潜在用户：对家族文化感兴趣的普通用户

**核心价值**：
- 数字化保存家族记忆，防止信息流失
- 可视化展示家族关系，增强家族凝聚力
- 便捷化管理家族事务，提升协作效率

## 二、核心功能模块

### 2.1 家族树系统 🌳

**功能描述**：可视化的家族关系图谱展示

**详细需求**：

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 树形展示 | 采用树状图展示家族成员关系，支持横向/纵向布局切换 | P0 |
| 缩放导航 | 支持放大、缩小、平移操作，快速定位到指定成员 | P0 |
| 关系标注 | 清晰标注父子、母子、夫妻、兄弟姐妹等关系 | P0 |
| 快速搜索 | 输入姓名快速定位到家族树中的位置 | P1 |
| 导出分享 | 支持导出为图片/PDF 格式，可设置查看权限分享给家族成员 | P1 |
| 时间轴视图 | 按年代展示家族发展历程，重大事件标记 | P2 |

**技术实现方案**：

```
前端可视化：
- 使用 D3.js v7 实现力导向图布局
- Canvas 渲染大规模节点（>500个节点时自动切换）
- 虚拟滚动优化性能

数据结构：
- 邻接表存储家族关系
- 使用 Redis 缓存热点家族树数据
- 支持增量加载（按代际懒加载）

算法优化：
- Sugiyama 算法优化层次布局
- 碰撞检测避免节点重叠
- 路径压缩算法快速查找关系链
```

**接口需求**：
- `GET /api/family-tree/:familyId` - 获取家族树数据
- `POST /api/family-tree/node` - 添加家族成员
- `PUT /api/family-tree/relation` - 更新关系
- `GET /api/family-tree/export` - 导出家族树

### 2.3 个人相册系统 📸

**功能描述**：多媒体家族记忆存储与展示

**存储方案**：

```
图片处理流程：
1. 客户端上传 → 2. 服务端接收 → 3. 格式验证 → 4. 压缩优化 → 5. OSS存储 → 6. CDN分发

压缩策略：
- 原图：保留原始文件
- 缩略图：200x200（列表展示）
- 中图：800x800（详情页）
- 大图：1920x1920（全屏查看）

存储结构：
/{familyId}/{userId}/{albumId}/{photoId}_original.jpg
/{familyId}/{userId}/{albumId}/{photoId}_thumb.jpg
/{familyId}/{userId}/{albumId}/{photoId}_medium.jpg
/{familyId}/{userId}/{albumId}/{photoId}_large.jpg
```

**AI功能集成**：
- 人脸识别：调用阿里云人脸识别API
- 老照片修复：对接腾讯云图像修复服务
- 照片上色：使用开源模型 DeOldify

**接口设计**：
- `POST /api/album/upload` - 上传照片（支持分片上传）
- `GET /api/album/:albumId/photos` - 获取相册照片列表
- `POST /api/photo/:photoId/restore` - 照片修复
- `POST /api/photo/:photoId/colorize` - 照片上色

## 三、用户体验流程

### 3.1 新用户注册流程

```
1. 访问首页 → 2. 点击注册 → 3. 选择注册方式（手机/微信）
→ 4. 填写基本信息 → 5. 创建/加入家族 → 6. 完善个人档案 → 7. 进入主页
```

**关键体验点**：
- 注册流程不超过3步
- 支持微信一键登录
- 首次登录引导教程（可跳过）

### 3.2 核心功能用户旅程

**场景1：族长创建家族树**
```
进入家族树 → 添加祖先节点 → 逐层添加后代 → 标注关系 → 邀请成员完善 → 导出分享
```

**场景2：成员上传家族照片**
```
进入相册 → 创建相册 → 批量上传照片 → AI识别人脸 → 标注人物 → 设置权限 → 发布
```

**场景3：组织家族活动**
```
发起活动 → 填写活动信息 → 邀请成员 → 成员报名 → 费用统计 → 活动提醒 → 活动回顾
```

### 3.3 交互设计原则

- **简洁优先**：核心功能3次点击内完成
- **容错设计**：重要操作二次确认
- **响应式**：适配手机、平板、PC
- **无障碍**：支持大字体、高对比度模式

## 五、数据库设计

### 5.1 用户与权限模块

**用户表 (users)**
```sql
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) UNIQUE NOT NULL COMMENT '用户名',
  phone VARCHAR(20) UNIQUE COMMENT '手机号',
  email VARCHAR(100) UNIQUE COMMENT '邮箱',
  password_hash VARCHAR(255) COMMENT '密码哈希',
  avatar_url VARCHAR(500) COMMENT '头像URL',
  nickname VARCHAR(50) COMMENT '昵称',
  real_name VARCHAR(50) COMMENT '真实姓名',
  gender ENUM('male', 'female', 'unknown') DEFAULT 'unknown',
  birth_date DATE COMMENT '出生日期',
  status TINYINT DEFAULT 1 COMMENT '状态：1-正常 0-禁用',
  last_login_at TIMESTAMP COMMENT '最后登录时间',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_phone (phone),
  INDEX idx_email (email),
  INDEX idx_username (username)
) COMMENT='用户表';
```

**用户角色表 (user_roles)**
```sql
CREATE TABLE user_roles (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  family_id BIGINT NOT NULL,
  role_type ENUM('admin', 'branch_admin', 'member', 'guest') DEFAULT 'member',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (family_id) REFERENCES families(id),
  UNIQUE KEY uk_user_family (user_id, family_id)
) COMMENT='用户角色表';
```

**权限表 (permissions)**
```sql
CREATE TABLE permissions (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  role_type VARCHAR(50) NOT NULL,
  resource VARCHAR(100) NOT NULL COMMENT '资源：family_tree, album, activity等',
  action VARCHAR(50) NOT NULL COMMENT '操作：create, read, update, delete',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_role_resource_action (role_type, resource, action)

```

```
POST /auth/login

{

```

## 八、风险管理与约束

### 8.1 技术风险

| 风险项 | 影响 | 概率 | 应对措施 |
|--------|------|------|----------|
| 大规模家族树渲染性能 | 高 | 中 | 虚拟滚动、分层加载、Canvas渲染 |
| 图片存储成本过高 | 中 | 高 | 智能压缩、CDN加速、冷热数据分离 |
| 数据库查询性能瓶颈 | 高 | 中 | 读写分离、分库分表、Redis缓存 |
| 第三方API依赖 | 中 | 低 | 多供应商备份、降级方案 |

- 用户数据加密存储
